# 主要修改：最终输出模块加入 NS Token 输入（Q + NS MeanPool 融合）

## 修改目的

当前模型在 `_run_multi_seq_blocks` 的最终输出阶段，仅使用了 Q tokens 的信息：

```python
all_q = torch.cat(curr_qs, dim=1)     # (B, Nq*S, D)
output = all_q.view(B, -1)            # (B, Nq*S*D)
output = self.output_proj(output)     # (B, D)
```

NS tokens (`curr_ns`) 经过每层 `MultiSeqHyFormerBlock` 中的 `RankMixerBlock` 处理后，蕴含了丰富的跨序列交互信息，但在最终输出时被**完全丢弃**。

本次修改将 NS tokens 的聚合表示与 Q tokens 一起输入到 `output_proj`，使 NS 信息参与最终预测，类似 BERT 的 `[CLS]` 设计。

## 问题定位

- **文件**：`model.py`
- **方法**：`PCVRHyFormer._run_multi_seq_blocks`
- **行号**：~1671-1675

```python
# 原代码：仅使用 Q tokens
B = curr_qs[0].shape[0]
all_q = torch.cat(curr_qs, dim=1)  # (B, Nq*S, D)
output = all_q.view(B, -1)         # (B, Nq*S*D)
output = self.output_proj(output)  # (B, D)
```

问题：`curr_ns` (shape: `(B, num_ns, D)`) 未参与最终输出。

## 修改方案：Q + NS MeanPool 融合

### 核心思路

将 NS tokens 做 Mean Pooling 得到聚合向量 `ns_out`，与 Q tokens 的展平向量拼接后，通过修改输入维度的 `output_proj` 投影到 `d_model` 维度。

```python
B = curr_qs[0].shape[0]
all_q = torch.cat(curr_qs, dim=1)                  # (B, Nq*S, D)

# NS aggregation: mean pool over all NS tokens
ns_out = curr_ns.mean(dim=1)                       # (B, D)

# Concatenate Q tokens and NS aggregation token
output = torch.cat([all_q.view(B, -1), ns_out], dim=-1)  # (B, (Nq*S+1)*D)
output = self.output_proj(output)                  # (B, D)
```

## 修改文件清单

### 1. `model.py` — `PCVRHyFormer.__init__`

**修改 `output_proj` 的输入维度：**

```python
# 原代码
self.output_proj = nn.Sequential(
    nn.Linear(num_queries * self.num_sequences * d_model, d_model),
    nn.LayerNorm(d_model),
)

# 修改为 (+1 表示额外加入 NS aggregation token 的维度)
self.output_proj = nn.Sequential(
    nn.Linear((num_queries * self.num_sequences + 1) * d_model, d_model),
    nn.LayerNorm(d_model),
)
```

### 2. `model.py` — `PCVRHyFormer._run_multi_seq_blocks`

**修改最终输出逻辑：**

```python
# 原代码（第 1671-1675 行）
# B = curr_qs[0].shape[0]
# all_q = torch.cat(curr_qs, dim=1)  # (B, Nq*S, D)
# output = all_q.view(B, -1)         # (B, Nq*S*D)
# output = self.output_proj(output)  # (B, D)

# 修改后
B = curr_qs[0].shape[0]
all_q = torch.cat(curr_qs, dim=1)                  # (B, Nq*S, D)

# NS aggregation: mean pool over all NS tokens
ns_out = curr_ns.mean(dim=1)                       # (B, D)

# Concatenate Q tokens and NS aggregation token
output = torch.cat([all_q.view(B, -1), ns_out], dim=-1)  # (B, (Nq*S+1)*D)
output = self.output_proj(output)                  # (B, D)
```

## 影响分析

| 方面 | 影响 |
|------|------|
| **参数量** | `output_proj` 输入维度从 `Nq*S*D` 变为 `(Nq*S+1)*D`，增加 `d_model` 个参数，增幅可忽略不计 |
| **训练/推理兼容性** | `train.py` 和 `infer.py` 完全无需修改，纯模型内部改动 |
| **已有 checkpoint** | **不兼容**，`output_proj.0.weight` 的 shape 从 `(D, Nq*S*D)` 变为 `(D, (Nq*S+1)*D)`，需要重新训练 |
| **计算开销** | 仅增加一个 mean pooling 操作和略微增大的 Linear 层，开销极小 |
| **RankMixer 约束** | 无需修改，`output_proj` 不在 `RankMixerBlock` 内，不受 `d_model % T == 0` 约束 |

## 设计决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| NS 聚合方式 | Mean Pooling | 简单高效，无额外参数；NS tokens 经过 RankMixer 后已充分交互，mean pool 能捕获整体语义 |
| 拼接位置 | 与 Q tokens 展平后拼接 | 保持与原有架构一致的线性投影范式，最小化改动 |
| 是否使用独立 NS Head | 否 | 先验证最简单的融合方式；若效果不明显，再升级为带门控的独立 Aggregation Head |
| 输出维度 | 保持 `d_model` | 与下游 `clsfier` 的输入维度一致，无需修改分类器结构 |

## 使用方式

无需额外 CLI 参数，修改后直接训练即可：

```bash
python train.py ...
```

## 后续可扩展方向

如果 MeanPool 融合效果良好，可进一步升级为以下方案：

1. **加权 MeanPool**：学习每个 NS token 的注意力权重，取代 uniform mean
2. **独立 NS Aggregation Head**：`ns_out = MLP(mean(curr_ns))`，让模型学习更复杂的 NS 聚合变换
3. **门控融合**：`output = gate * q_out + (1-gate) * ns_out`，显式控制 Q 和 NS 的混合比例
