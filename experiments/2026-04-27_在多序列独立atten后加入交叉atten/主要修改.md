# 主要修改：多序列独立 Attention 后加入交叉 Attention

## 修改目的

原 `MultiSeqHyFormerBlock` 中，4 个 Domain Sequence（seq_a / seq_b / seq_c / seq_d）各自独立编码、独立 Cross-Attention，序列之间**没有显式信息交换**。多个序列的 Query tokens 仅在最后拼接后经 RankMixer 做隐式交互，信息融合不充分。

本次修改在 Independent Query Decoding 之后、Token Fusion 之前，插入**序列间 Cross-Attention（Inter-Sequence Cross-Attention）**，让每个序列的 Q tokens 显式关注其他所有序列的编码后 tokens，实现跨域信息交换。

## 修改后的前向流程

```
Step 1: Independent Sequence Evolution  （不变）
    每个序列独立通过各自的 seq_encoder

Step 2: Independent Query Decoding  （不变）
    每个序列的 Q token 独立 cross-attend 本序列编码结果

Step 2.5: Inter-Sequence Cross-Attention  （新增）
    对每个序列 i：
      KV = concat(其他所有序列 j≠i 的编码 tokens)  → (B, sum(L_j), D)
      Q_i = inter_seq_attns[i](decoded_q_i, KV, mask)
      → 序列 i 的 Q token 获得了其他序列的上下文信息

Step 3: Token Fusion  （不变）
    拼接所有 decoded_q + ns_tokens

Step 4: Query Boosting (RankMixer)  （不变）
    联合 mixing
```

## 修改文件清单

### 1. `model.py` — `MultiSeqHyFormerBlock`

**`__init__` 新增参数与模块：**

- `use_inter_seq_attn: bool = False` — 开关参数，默认关闭
- 当 `use_inter_seq_attn=True` 时，创建 `self.inter_seq_attns`：`nn.ModuleList` 包含 S 个独立的 `CrossAttention` 层（每个序列一个，非共享权重）

**`forward` 新增 Step 2.5：**

```python
if self.use_inter_seq_attn:
    inter_decoded_qs = []
    for i in range(S):
        # 拼接所有其他序列的 tokens 和 masks
        other_seqs = [next_seqs[j] for j in range(S) if j != i]
        other_masks = [next_masks[j] for j in range(S) if j != i]
        inter_kv = torch.cat(other_seqs, dim=1)    # (B, sum(L_j), D)
        inter_mask = torch.cat(other_masks, dim=1) # (B, sum(L_j))
        # Q_i attend 其他所有序列
        inter_q_i = self.inter_seq_attns[i](
            decoded_qs[i], inter_kv, inter_mask,
            rope_cos=None, rope_sin=None,
        )
        inter_decoded_qs.append(inter_q_i)
    decoded_qs = inter_decoded_qs
```

### 2. `model.py` — `PCVRHyFormer`

- `__init__` 新增 `use_inter_seq_attn: bool = False` 参数，存储并透传到 `MultiSeqHyFormerBlock`

### 3. `train.py`

- 新增 CLI 参数 `--use_inter_seq_attn`（`action='store_true'`，默认 `False`）
- 在 `model_args` 字典中添加 `"use_inter_seq_attn": args.use_inter_seq_attn`

### 4. `infer.py`

- 在 `_FALLBACK_MODEL_CFG` 中添加 `'use_inter_seq_attn': False`，确保推理时兼容旧 checkpoint

## 使用方式

```bash
# 默认行为（关闭，与原模型完全一致）
python train.py ...

# 开启序列间交叉注意力
python train.py ... --use_inter_seq_attn
```

## 设计决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 插入位置 | Step 2 之后、Step 3 之前 | Q token 先充分理解本序列（Step 2），再去关注其他序列（Step 2.5） |
| KV 来源 | 其他序列的 **编码后 tokens** (next_seqs) | 比原始未编码 tokens 语义更丰富 |
| 参数共享 | 每个序列独立 CrossAttention | 表达能力更强，不同序列关注其他序列的侧重点可能不同 |
| RoPE | 不使用 | 拼接后的序列来自不同 domain，长度/语义差异大，统一位置编码意义有限 |
| 残差连接 | CrossAttention 内部已包含 | `residual + attn_out`，无需额外处理 |
| 开关设计 | bool 参数 + CLI flag | 方便 A/B 对比实验，关闭时行为完全不变 |

## 参数量影响

以 `d_model=64, num_heads=4, S=4` 为例：

- 每个 `CrossAttention` 的参数量：`4 * d_model^2 = 4 * 64 * 64 = 16,384`
- 新增总参数量：`S * 16,384 = 65,536`（约 65K，相对整体模型参数量增幅很小）
- 每个 `MultiSeqHyFormerBlock` 增加一层，`num_hyformer_blocks=2` 时总增加约 130K 参数
