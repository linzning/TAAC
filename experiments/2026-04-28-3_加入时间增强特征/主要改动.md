# P0：时间感知增强特征

## 改动概述

在原有 `time_bucket`（粗粒度分桶）基础上，增加精细时间特征与全局时间特征，并将这些特征通过投影层融入模型。

---

## 改动文件

- `dataset.py`
- `model.py`
- `trainer.py`
- `infer.py`

---

## 1. dataset.py —— 新增时间特征计算

### 每个序列域的精细时间特征

在 `_convert_batch` 的各 domain 循环内，基于已有时间戳数据计算：

| 特征名 | 计算方式 | 维度 | 说明 |
|--------|---------|------|------|
| `{domain}_last_action_diff` | `timestamp - 序列最近一次行为时间戳` | (B,) | 用户在该域的活跃度/沉默度 |
| `{domain}_time_gap_mean` | 序列内相邻行为时间差均值 | (B,) | 行为规律性 |
| `{domain}_time_gap_std` | 序列内相邻行为时间差标准差(ddof=0) | (B,) | 行为规律性波动（<2个间隙时置0） |

**计算细节**：
- 利用 `ts_padded`（已 padding 到 max_len 的时间戳序列）与 `lengths`（实际长度）计算
- 仅当 `seq_len >= 2` 时计算 `mean`/`std`；`seq_len == 1` 时只有 `last_action_diff`
- 相邻时间差通过 `np.diff(valid_ts)` 计算，并用 `np.maximum(..., 0)` 保证非负

### 全局时间特征

在循环结束后计算：

| 特征名 | 计算方式 | 维度 |
|--------|---------|------|
| `hour_of_day` | `(timestamp % 86400) // 3600` | (B,) |
| `day_of_week` | `(timestamp // 86400) % 7` | (B,) |

两者 stack 为 `global_time_feats` (B, 2) 输出。

## 2. model.py —— 模型侧特征融合

### ModelInput 扩展

新增两个字段：
- `global_time_feats: torch.Tensor`  # [B, 2]
- `seq_time_stats: dict`             # {domain: [B, 3]}

### 新增投影层

```python
# 全局时间特征投影：2 -> d_model
global_time_proj = nn.Sequential(nn.Linear(2, d_model), nn.LayerNorm(d_model))

# 各域序列时间统计投影：3 -> d_model（每个 domain 独立）
seq_time_stat_projs[domain] = nn.Sequential(nn.Linear(3, d_model), nn.LayerNorm(d_model))
```

### 融合方式

1. **序列级时间统计**：在 `_embed_seq_domain` 中，将 `time_stats` (B, 3) 投影为 `(B, 1, D)`，以广播形式加到序列 token embedding 上，使该域的每个 token 都感知到用户在此域的时间行为模式。

2. **全局时间特征**：在 `forward`/`predict` 的 `_run_multi_seq_blocks` 输出后，将 `global_time_feats` 投影为 `(B, D)`，作为残差偏置加到最终输出上。

**注意**：此设计不改变 `num_ns` 和 `T`，因此不影响 `rank_mixer_mode='full'` 时的整除约束。

---

## 3. trainer.py / infer.py —— ModelInput 构造适配

在 `_make_model_input` / `_batch_to_model_input` 中：
- 将各域的 `last_action_diff`、`time_gap_mean`、`time_gap_std` stack 为 `(B, 3)` 填入 `seq_time_stats[domain]`
- 将 `global_time_feats` (B, 2) 直接传入
- 若某 key 不存在则回退到全零张量，保持向后兼容

---

## 特征作用总结

| 特征类别 | 特征名 | 作用 |
|---------|--------|------|
| 序列级 | `last_action_diff` | 反映用户在该域的活跃/沉默程度 |
| 序列级 | `time_gap_mean` | 反映行为发生的平均间隔规律 |
| 序列级 | `time_gap_std` | 反映行为间隔的稳定性 |
| 全局 | `hour_of_day` | 捕捉日内时段偏好 |
| 全局 | `day_of_week` | 捕捉周内周期偏好 |
