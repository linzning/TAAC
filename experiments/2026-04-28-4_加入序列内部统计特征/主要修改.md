# 主要修改：序列内部统计特征

## 修改目的

原模型仅利用序列的 Embedding 表示（通过 item_id 及 side-info 的 Embedding + Attention）来捕获用户行为模式，缺乏对**序列结构本身**的显式建模。例如：

- 同一序列中 item 的重复程度（`unique_ratio`、`repeat_count`）
- 用户行为的集中度（`top_freq`）
- 跨域行为的分布差异（`active_domain_count`、`max_domain_ratio`）

这些信息不依赖任何特征语义，纯粹从序列结构即可计算，且可能蕴含丰富的用户行为模式信号。

本次修改在模型 `forward` 阶段实时计算上述统计特征，投影为 `d_model` 维向量后，以残差方式融入最终输出，辅助分类决策。

## 新增特征定义

### 序列内部统计特征（每域独立）

| 特征名 | 计算方式 | 说明 |
|--------|---------|------|
| `{domain}_seq_len` | 已有 `seq_lens[domain]` | 序列有效长度 |
| `{domain}_unique_ratio` | 唯一 item 数 / seq_len | 行为多样性，越低表示重复行为越多 |
| `{domain}_repeat_count` | seq_len - 唯一 item 数 | 重复行为次数 |
| `{domain}_top_freq` | 最频繁 item 出现次数 / seq_len | 行为集中度，越高表示越集中在少数 item |

### 跨序列聚合特征（全局）

| 特征名 | 计算方式 | 说明 |
|--------|---------|------|
| `total_seq_len` | 4 个 domain seq_len 之和 | 总行为量 |
| `active_domain_count` | seq_len > 0 的 domain 数 | 行为覆盖广度 |
| `max_domain_ratio` | max(seq_len) / total_seq_len | 行为集中域占比 |

## 修改文件清单

### 1. `model.py` — `PCVRHyFormer`

**`__init__` 新增参数与模块：**

- `use_seq_stats: bool = True` — 开关参数，默认开启
- 当 `use_seq_stats=True` 时，创建 `self.seq_stats_proj`：
  - 输入维度：`4 * num_sequences + 3`
  - 输出维度：`d_model`
  - 结构：`Linear → LayerNorm → SiLU → Dropout`

**新增 `_compute_seq_stats` 方法：**

```python
@torch.no_grad()
def _compute_seq_stats(self, seq_data: dict, seq_lens: dict) -> torch.Tensor:
```

计算逻辑：
1. 对每个 domain，取 `seq_data[domain][:, 0, :]` 作为序列主 ID（item_id）
2. 根据 `seq_lens` 生成 padding mask，仅对有效位置统计
3. 逐 sample 计算 `unique_ratio`、`repeat_count`、`top_freq`
4. 跨域聚合 `total_seq_len`、`active_domain_count`、`max_domain_ratio`
5. 返回 `[B, 4*S + 3]` 的统计特征张量

**`forward` / `predict` 融入逻辑：**

在 `_run_multi_seq_blocks` 返回 `output` 后、`clsfier` 前插入：

```python
if self.use_seq_stats:
    seq_stats = self._compute_seq_stats(inputs.seq_data, inputs.seq_lens)
    stats_emb = self.seq_stats_proj(seq_stats)  # [B, d_model]
    output = output + stats_emb                 # residual-like 融合
```

### 2. `train.py`

- 新增 CLI 参数 `--use_seq_stats`（默认开启）与 `--no_seq_stats`（用于关闭）
- 在 `model_args` 字典中添加 `"use_seq_stats": args.use_seq_stats`

### 3. `infer.py`

- 在 `_FALLBACK_MODEL_CFG` 中添加 `'use_seq_stats': True`，确保推理兼容

## 设计决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 计算位置 | 模型 `forward` 内实时计算 | 不改数据管道，infer 无需同步 `dataset.py`；逻辑内聚 |
| 主 ID 来源 | `seq_data[domain][:, 0, :]` | schema 中每域第 1 个 feature 为 item_id，符合当前数据结构 |
| unique 计算方式 | 逐 sample `torch.unique` | PyTorch 无 batch-wise unique；B=256、L<=512，计算开销可忽略 |
| 融入方式 | `output = output + stats_emb` | 不增加 T（不影响 RankMixer 约束），不改变序列长度，残差相加最轻量 |
| 开关设计 | `--use_seq_stats` / `--no_seq_stats` | 便于消融实验，默认开启 |
| 梯度传播 | `@torch.no_grad()` 包裹统计计算 | `torch.unique` 不可微；投影层参数仍可学习 |

## 使用方式

```bash
# 默认行为（开启序列统计特征）
python train.py ...

# 关闭序列统计特征（消融实验）
python train.py ... --no_seq_stats
```

## 参数量影响

以 `d_model=64, S=4` 为例：

- `seq_stats_proj` 输入维度：`4 * 4 + 3 = 19`
- `seq_stats_proj` 参数量：`19 * 64 + 64 = 1,280`
- 相对于整体模型参数量增幅极小（通常 < 0.01%）

## 与已有实验的关系

本实验基于 `2026-04-28-3_加入时间增强特征` 的代码基线，在其基础上进一步增加**序列结构统计特征**，与时间特征形成互补：

- **时间特征**：刻画行为发生的时间规律（when）
- **统计特征**：刻画行为的结构模式（how diverse / how concentrated）
